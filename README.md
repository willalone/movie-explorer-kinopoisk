## Movie Explorer (тестовое задание)

Небольшое многостраничное SPA для просмотра фильмов. Сделано на **React + TypeScript** и использует API Кинопоиска (PoiskKino / kinopoisk.dev).

Если коротко, то тут:
- список фильмов с бесконечной прокруткой
- фильтры (жанры/рейтинг/год) с синхронизацией в `searchParams`
- страница с деталями фильма
- избранное (с подтверждением в модалке и сохранением в `localStorage`)
- сравнение до 2 фильмов (таблица + возможность свернуть)

### Стек

- **React 19**
- **TypeScript**
- **Vite**
- **react-router-dom v7**
- **Axios**

### Возможности

1. **Список фильмов**
   - подгрузка по `50` фильмов за запрос
   - infinite scroll через `IntersectionObserver`
   - карточка: постер (если есть), название, год, рейтинг и жанры

2. **Фильтрация**
   - жанры: множественный выбор
   - рейтинг: диапазон
   - год выпуска: диапазон начиная с `1990`
   - значения синхронизируются с URL (так что прямые ссылки с фильтрами работают)
   - добавлены понятные подсказки об ошибках для некорректных диапазонов

3. **Детальная страница**
   - постер (или плейсхолдер “Нет постера”)
   - название, описание, рейтинг
   - дата выхода (если доступна)
   - список жанров и длительность (если есть)

4. **Избранное**
   - отдельная страница
   - хранится в `localStorage` (не пропадает после перезагрузки)
   - добавление и удаление — только после подтверждения в кастомном модальном окне

5. **Сравнение**
   - до 2 фильмов
   - если выбрать третий — первый автоматически сбрасывается
   - в сравнительной области: год, рейтинг, жанры, длительность
   - есть кнопка свернуть/развернуть таблицу

### Настройка окружения

1. Создайте `.env` на основе `.env.default`:

```bash
cp .env.default .env
```

2. Укажите API ключ:

```bash
VITE_KINOPOISK_API_KEY=your_real_api_key
```

Документация: https://kinopoisk.dev/

### Структура проекта (по смыслу)

- `src/api` — запросы и маппинг данных (`axios`, DTO → доменная модель)
- `src/types` — типы для списка/деталей
- `src/pages` — страницы (`MoviesPage`, `MovieDetailsPage`, `FavoritesPage`)
- `src/components` — UI-компоненты (карточки, фильтры, infinite scroller, модалка, панель сравнения)
- `src/store` — состояние:
  - `FavoritesProvider` + `useFavorites`
  - `CompareProvider` + `useCompare`

### Решения и компромиссы

- **Фильтры в URL**: проще делиться ссылками и повторять результат.
- **Производительность**: запросы отменяются (AbortController), список грузится пачками, есть кеш на уровне API.
- **A11y**: модальное окно поддерживает `Esc`, делает focus management и возвращает фокус после закрытия.
