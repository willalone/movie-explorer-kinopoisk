## Movie Explorer (Тестовое задание)

Приложение для просмотра информации о фильмах c использованием **React**, **TypeScript** и API Кинопоиска (PoiskKino/kinopoisk.dev).

### Стек

- **React 18**, функциональные компоненты и хуки
- **TypeScript**
- **Vite**
- **React Router v6**
- **Axios**

### Возможности

- **Список фильмов** с бесконечной прокруткой (по 50 фильмов за запрос)
- **Фильтры**:
  - по жанрам (множественный выбор)
  - по рейтингу (диапазон)
  - по году выпуска, начиная с 1990 года (диапазон)
- **Синхронизация фильтров с URL** через `search parameters` — при переходе по ссылке фильтры восстанавливаются
- **Детальная страница фильма**:
  - постер
  - название
  - описание
  - рейтинг
  - дата выхода (мировая премьера, если доступна)
  - список жанров
- **Избранное**:
  - отдельная страница со списком избранных фильмов
  - данные хранятся в `localStorage`, не пропадают при перезагрузке
  - добавление в избранное только после подтверждения в кастомном модальном окне
- **Режим сравнения**:
  - выбор до двух фильмов для сравнения
  - при выборе третьего — первый автоматически сбрасывается
  - сравнительная таблица: название, год, рейтинг, жанры, длительность

### Настройка окружения

1. Создайте файл `.env` в корне проекта на основе `.env.default`:

```bash
cp .env.default .env
```

2. Впишите ваш API‑ключ Кинопоиска:

```bash
VITE_KINOPOISK_API_KEY=your_real_api_key
```

Документация по API: `https://kinopoisk.dev/`.

### Локальный запуск

```bash
npm install
npm run dev
```

Приложение будет доступно по адресу `http://localhost:5173/` (порт может отличаться).

### Сборка для продакшена

```bash
npm run build
npm run preview
```

### Структура ключевых модулей

- `src/api` — работа с API (`axios`, маппинг DTO → доменные модели)
- `src/types` — типы для фильмов
- `src/pages` — страницы приложения:
  - `MoviesPage` — список фильмов, фильтры, бесконечный скролл
  - `MovieDetailsPage` — детали фильма
  - `FavoritesPage` — избранное
- `src/components` — переиспользуемые компоненты:
  - карточка фильма, фильтры, панель сравнения, модальное окно подтверждения, обёртка бесконечного скролла
- `src/store` — контексты:
  - избранное (`FavoritesContext`) с сохранением в `localStorage`
  - сравнение фильмов (`CompareContext`)

### Решения и компромиссы

- **Фильтры в URL**: состояние фильтров синхронизируется через query‑параметры, поэтому страницу можно открывать по прямой ссылке и получать тот же список.
- **Производительность**: запросы к списку фильмов дебаунсятся, устаревшие запросы отменяются, в памяти держится краткоживущий кэш страниц.
- **A11y**: модальное окно поддерживает `Esc`, удержание фокуса внутри и возвращает фокус на элемент, который его открыл.

### Storybook

Для просмотра компонентов в отрыве от страниц:

```bash
npm run storybook
```

В Storybook вынесены карточка фильма, модальное окно подтверждения и панель сравнения.
